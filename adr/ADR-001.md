# ADR-001: [Separación del modelo en dominios funcionales]

## Contexto
El sistema maneja múltiples áreas funcionales como clientes, vuelos, pagos, seguridad y facturación.

## Problema
Un modelo monolítico dificulta la comprensión, mantenimiento y evolución del sistema. 

## Decisión
Se organiza la base de datos en dominios funcionales independientes, agrupando tablas según su responsabilidad (geografía, identidad, operaciones, ventas, etc.).

## Justificación técnica
Mejora la mantenibilidad del sistema 
Facilita la escalabilidad
Permite trabajo modular por equipos
Mejora la comprensión del modelo
